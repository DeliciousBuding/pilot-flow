# PilotFlow 用户指南

> 本文档面向**已经装好 PilotFlow** 的用户。不是安装指南，请先完成 [INSTALL.md](../INSTALL.md)。

## 我能问什么？

PilotFlow 是飞书群里的 AI 项目运行官，**你只管说话，Hermes Agent 帮你理解意图，PilotFlow 帮你执行飞书动作**。以下是真实可直接使用的指令样例。

---

## 一、创建项目

### 基础创建
```
@PilotFlow 帮我准备答辩项目空间，成员小王、小李，交付物是项目简报和任务清单，5月7日截止
```

### 不指定成员
```
@PilotFlow 帮我建一个活动策划项目，下周三截止
```
PilotFlow 会追问成员信息，不会瞎猜。

### 一句话建项目
```
@PilotFlow 帮我建个项目
```
PilotFlow 发现信息不全，会提示你补充目标、成员、交付物和截止时间。

### sprint 项目
```
@PilotFlow 帮我建一个两周 sprint，目标是完成核心功能
```

---

## 二、查看状态

### 项目看板
```
@PilotFlow 项目进展如何？
@PilotFlow 看看项目进展
```

### 筛选查看
```
@PilotFlow 看看未完成项目
@PilotFlow 看看逾期项目
@PilotFlow 看看风险项目
@PilotFlow 小王负责哪些项目？
```

### 站会简报
```
@PilotFlow 发一份站会简报
@PilotFlow 给我项目汇总
```

---

## 三、多轮管理

### 修改截止时间
```
@PilotFlow 把答辩项目的截止时间改成5月10日
```

### 新增成员
```
@PilotFlow 把小陈加到答辩项目
```
支持飞书 @ 提及：直接在群里 @ 小陈，系统会解析为项目成员。

### 移除成员（需确认）
```
@PilotFlow 把小李从答辩项目移除
```
⚠️ 移除成员属于高危动作，PilotFlow 会要求你先确认再执行。

### 新增交付物
```
@PilotFlow 给答辩项目新增交付物：验收报告
```
带负责人：```
@PilotFlow 给答辩项目新增交付物：小李负责验收报告
```

### 上报风险
```
@PilotFlow 答辩项目 API 审批可能卡住
```
PilotFlow 会把项目切到风险状态，同步更新飞书文档和多维表格。

### 解除风险
```
@PilotFlow 答辩项目风险解除了
```

## 四、卡片操作

PilotFlow 发到群里的互动卡片支持直接点击操作：

| 卡片类型 | 可点击按钮 | 说明 |
|---|---|---|
| 执行计划 | ✅ 确认执行 / ❌ 取消 | 确认后创建项目；取消后不创建 |
| 项目看板 | 查看状态 / 标记完成 / 发送提醒 / 创建待办 | 一键操作，原卡片状态实时更新 |
| 项目详情 | 标记完成 / 重新打开 / 解除风险 / 发送提醒 / 创建待办 | 按项目当前状态自动调整按钮 |
| 站会简报 | 查看风险 / 催办逾期 / 创建待办 | 支持按负责人筛选的批量操作 |
| 催办看板 | 一键催办 / 创建待办 | 按负责人筛选后批量操作 |

### 按钮失败自动重试
卡片操作如果飞书 API 临时失败，原卡片会显示"操作失败"并保留按钮。等网络恢复后点击同一按钮可重试，不需要重新发起。

---

## 五、文本确认 / 取消

如果卡片没出来或者你更习惯用文字：

```
确认执行          # 确认创建项目
取消              # 取消当前操作
```

⚠️ "确认卡片" / "确认计划" / "给我确认卡片" **不会**被当作执行确认，PilotFlow 能区分"要卡片"和"要执行"。

## 六、群聊订阅模式（进阶）

PilotFlow 默认需要 @ 才触发。开启"订阅模式"后，**群里所有消息都会进入 PilotFlow**，当 Hermes Agent 识别到目标 / 承诺 / 风险 / 行动项时，主动冒泡"要不要整理成项目？"卡片。

### 配置方式

编辑 `~/.hermes/config.yaml`，给目标群加 `require_mention: false`：

```yaml
feishu:
  group_rules:
    oc_你的群chat_id:
      policy: open
      require_mention: false
```

重启 gateway 生效。

### 适用场景
- **课程项目群**：讨论散落时 PilotFlow 主动建议项目化
- **站会群**：每天早上识别新承诺，建议更新项目状态
- **小团队协作**：从聊天直接沉淀交付物和截止时间

### 不适用场景
- 纯闲聊群（不建议开启，成本高且会刷屏）
- 已有完善飞书项目体系的群（数据冲突）

## 七、常见问题

### Bot 没回复怎么办？
1. 检查 `hermes gateway` 是否在运行
2. 确认 `.env` 里 `FEISHU_APP_ID` / `FEISHU_APP_SECRET` 正确
3. 群里 @ 的是 Bot 机器人，不是其他成员
4. 重启 gateway 试试

### 计划卡片出不来？
1. `python setup.py --hermes-dir <hermes-path> --hermes-home ~/.hermes` 重新安装
2. `scripts/verify_wsl_feishu_runtime.py --probe-llm` 检查 LLM 连接（401 排查）

### 确认执行后 Bot 报错？
PilotFlow 会自动重试飞书 API 失败的卡片操作。如果持续失败：
1. 确认飞书应用权限（im, docx, bitable, task, drive, calendar）
2. `pilotflow_health_check` 查看运行状态

### PilotFlow 重启后我的项目还在吗？
是的。项目摘要（状态、交付物、截止时间、最近进展）存入脱敏本地状态文件，Hermes 重启后自动恢复。
