# PilotFlow 创新点与功能规划

## 核心创新（评委关注）

### 1. 群聊即项目入口（已完成）
- 飞书群里 @PilotFlow 一句话启动项目
- 不需要打开其他工具，不需要手动配置
- 入口最自然，使用门槛最低

### 2. AI 理解意图 + 真实执行（已完成）
- LLM 理解中文自然语言，提取目标/成员/交付物/截止时间
- 不是生成文字，而是调用飞书 API 创建真实产物
- 文档、表格、任务、消息，全部自动化

### 3. 人类始终有控制权（已完成）
- 确认门控：先展示计划，确认后才执行
- 展示确认卡片 + 文字确认，每步可审查
- 不是黑盒，每步可审查

### 4. 权限自管理（已完成）
- 创建的文档自动开放链接访问
- 群成员自动加为编辑者
- 不需要手动分享、手动开权限

### 5. @mention 深度集成（已完成）
- 文档内 @提及成员（飞书 mention_user 元素）
- 消息中 @提及成员（<at> 标签）
- 自动解析群成员列表，名称 → open_id

### 6. 即插即用插件架构（已完成）
- 基于 Hermes Agent 运行时
- `python setup.py --hermes-dir ...` 安装，不修改 Hermes 代码
- 文本消息通过 Hermes registry.dispatch 走已有渠道，互动卡片通过 Feishu IM API 直发
- 卡片按钮通过插件级 `/card` 桥接处理，不需要 fork Hermes
- Doc/Task/Bitable 用 lark_oapi SDK 直连

### 7. 多轮项目管理（已完成）
- 用户可以改截止时间：「把答辩项目的截止时间改成5月10日」
- 用户可以查状态：「答辩项目进展如何？」→ 发送项目看板卡片
- 用户可以加成员：「把张三加到答辩项目」→ @提及新成员
- 内存项目注册表 + 多维表格同步更新（双写）
- 模糊匹配项目名称，智能定位目标项目

### 8. 智能模板库（已完成）
- 内置常见模板：答辩、sprint、活动策划、产品上线
- Hermes 负责理解用户意图并选择合适工具；PilotFlow 只保留执行层模板、确认门控和飞书落地能力，避免用关键词匹配冒充智能
- 建议默认交付物和工期

### 9. 风险预警（已完成）
- 自动检测：成员缺失、截止时间模糊、交付物不明确
- 返回风险等级和处理建议

### 10. 日历集成（已完成）
- 自动创建截止时间日历事件（UTC+8 时区，9:00 AM，1小时时长）
- 项目创建时同步创建日历提醒

### 11. 项目仪表盘（已完成）
- 查询项目状态，生成看板卡片
- 发送飞书卡片到群聊

### 12. Hermes Memory 写入（已完成）
- 项目创建时 best-effort 保存模式到 Hermes memory
- 通过 registry.dispatch("memory") 深度融合 Hermes 生态
- 默认只保存成员数量，不保存成员姓名；需要完整成员名时显式开启
- 目前已完成写入链路；读取历史项目并自动建议成员/交付物属于下一轮

## 评委版创新总结

| 维度 | 一句话 |
| --- | --- |
| 业务创新 | 把群聊里的临时讨论变成可确认、可执行、可追踪的项目启动流程。 |
| 技术创新 | 用 Hermes 的 LLM/插件/网关/memory/cron 做 Agent 底座，用飞书 OpenAPI 编排文档、表格、任务、卡片和权限。 |
| 可信控制 | 所有副作用经过确认门控，运行结果可追溯，权限和 @mention 自动处理，失败时可降级。 |
