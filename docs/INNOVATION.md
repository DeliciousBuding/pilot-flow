# PilotFlow 创新点与功能规划

## 核心创新（评委关注）

### 1. 群聊即项目入口（已完成）
- 飞书群里 @PilotFlow 一句话启动项目
- 不需要打开其他工具，不需要手动配置
- 入口最自然，使用门槛最低

### 2. AI 理解意图 + 真实执行（已完成）
- LLM 理解中文自然语言，提取目标/成员/交付物/截止时间
- 不是生成文字，而是调用飞书 API 创建真实产物
- 文档、表格、任务、消息，全部自动化

### 3. 人类始终有控制权（已完成）
- 确认门控：先展示计划，确认后才执行
- 互动卡片 + 按钮，一键确认
- 不是黑盒，每步可审查

### 4. 权限自管理（已完成）
- 创建的文档自动开放链接访问
- 群成员自动加为编辑者
- 不需要手动分享、手动开权限

### 5. @mention 深度集成（已完成）
- 文档内 @提及成员（飞书 mention_user 元素）
- 消息中 @提及成员（<at> 标签）
- 自动解析群成员列表，名称 → open_id

### 6. 即插即用插件架构（已完成）
- 基于 Hermes Agent 运行时
- cp -r 安装，不修改 Hermes 代码
- 消息通过 Hermes registry.dispatch 走已有渠道
- Doc/Task/Bitable 用 lark_oapi SDK 直连

## 待实现创新（提升竞争力）

### 7. 多轮项目管理
- 用户可以改截止时间：「把答辩项目的截止时间改成5月10日」
- 用户可以查状态：「答辩项目进展如何？」
- 用户可以加成员：「把张三加到答辩项目」
- PilotFlow 查询 bitable 记录，更新对应字段

### 8. 项目模板库
- 内置常见模板：答辩、sprint、活动策划、产品上线
- 用户说「用答辩模板创建项目」自动填充默认交付物
- 模板可自定义，存储在 bitable 中

### 9. 风险预警与裁决
- 自动检测：成员缺失、截止时间模糊、交付物不明确
- 发送风险裁决卡到群，用户选择处理方式
- 记录风险决策到 bitable

### 10. 日历集成
- 自动创建截止时间日历事件
- 项目成员自动加入日历
- 临近截止时间自动提醒

### 11. 审批流集成
- 项目计划走飞书审批
- 变更走审批，有审计记录
- 管理员一键审批

### 12. Hermes Memory 集成
- 记住用户的常用成员、常用交付物
- 记住项目的成功模式
- 越用越聪明

### 13. 项目仪表盘
- 查询 bitable 生成项目状态摘要
- 按状态筛选：进行中、已完成、有风险
- 群内发送项目看板

### 14. 批量操作
- 一次创建多个子项目
- 批量分配任务
- 批量更新状态

## 竞赛亮点总结

| 维度 | 亮点 |
| --- | --- |
| 完整性 | 一句话→文档+表格+任务+消息+权限，全链路自动化 |
| 创新性 | 群聊即项目入口，AI 理解意图+真实执行，确认门控 |
| 技术深度 | Hermes 插件架构，lark_oapi SDK，@mention 解析，权限自管理 |
| 用户体验 | 30秒完成，中文输出，不暴露技术细节 |
| 可扩展性 | 即插即用，多轮管理，模板库，记忆进化 |
