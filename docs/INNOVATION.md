# PilotFlow 创新点与功能规划

## 核心创新（评委关注）

### 1. 群聊即项目入口（已完成）
- 飞书群里 @PilotFlow 一句话启动项目
- 不需要打开其他工具，不需要手动配置
- 入口最自然，使用门槛最低

### 2. AI 理解意图 + 真实执行（已完成）
- LLM 理解中文自然语言，提取目标/成员/交付物/截止时间
- 不是生成文字，而是调用飞书 API 创建真实产物
- 文档、表格、任务、消息，全部自动化

### 3. 人类始终有控制权（已完成）
- 确认门控：先展示计划，确认后才执行
- 展示确认卡片 + 文字确认，每步可审查
- 不是黑盒，每步可审查

### 4. 权限自管理（已完成）
- 创建的文档自动开放链接访问
- 群成员自动加为编辑者
- 不需要手动分享、手动开权限

### 5. @mention 深度集成（已完成）
- 文档内 @提及成员（飞书 mention_user 元素）
- 消息中 @提及成员（<at> 标签）
- 自动解析群成员列表，名称 → open_id

### 6. 即插即用插件架构（已完成）
- 基于 Hermes Agent 运行时
- `python setup.py --hermes-dir ...` 安装，不修改 Hermes 代码
- 文本消息通过 Hermes registry.dispatch 走已有渠道，互动卡片通过 Feishu IM API 直发
- 卡片按钮通过插件级 `/card` 桥接处理，不需要 fork Hermes
- Doc/Task/Bitable 用 lark_oapi SDK 直连

### 7. 多轮项目管理（已完成）
- 用户可以改截止时间：「把答辩项目的截止时间改成5月10日」
- 用户可以查状态：「答辩项目进展如何？」→ 发送项目看板卡片
- 用户可以加成员：「把张三加到答辩项目」→ @提及新成员
- 内存项目注册表 + 多维表格同步更新（双写）
- 模糊匹配项目名称，智能定位目标项目

### 8. 智能模板参考（已完成）
- 内置参考模板：答辩、sprint、活动策划、产品上线
- Agent 显式传入 template 参数，PilotFlow 只展示参考建议，不替 Agent 补全字段
- 缺字段时返回 needs_clarification，由 Agent 追问用户

### 9. 风险预警（已完成）
- 自动检测：成员缺失、截止时间模糊、交付物不明确
- 返回风险等级和处理建议

### 10. 日历集成（已完成）
- 自动创建截止时间日历事件（UTC+8 时区，9:00 AM，1小时时长）
- 项目创建时同步创建日历提醒

### 11. 项目仪表盘（已完成）
- 查询项目状态，生成看板卡片
- 发送飞书卡片到群聊

### 12. Hermes Memory 写入（已完成）
- 项目创建时 best-effort 保存模式到 Hermes memory
- 通过 registry.dispatch("memory") 深度融合 Hermes 生态
- 默认只保存成员数量，不保存成员姓名；需要完整成员名时显式开启
- 目前已完成写入链路；读取历史项目并自动建议成员/交付物属于下一轮

### 13. Agent 主驾驶——工具不拍板（已完成）
- 5 类语义参数（view_mode / template / risk_level / page / filters）必须 Agent 显式传入
- 工具默认拒绝从用户原话关键词推断，返回 needs_clarification 让 Agent 追问
- `allow_inferred_*=true` 仅作为旧版兼容路径，schema 标注"仅供回归测试"
- 保证"谁在思考"和"谁在执行"边界清晰

### 14. 确认 token + idempotency key（已完成）
- 每次计划生成产生独立 confirm_token 和 idempotency_key
- 重复确认或重复按钮点击不会重复创建飞书产物
- 过期 token 自动拒绝，防止过期操作误执行

### 15. opaque card action ref（已完成）
- 卡片按钮不携带可伪造的项目名/chat_id，只携带短期 opaque action id
- 11 类高风险操作（标记完成/重开/催办/待办/分页等）强制从 action ref 解析
- 防止伪造 `{"pilotflow_action": "mark_project_done"}` 直灌处理器

### 16. 卡片失败自动重试（已完成）
- 7 类卡片操作失败后保留 action ref，网络恢复可点同一按钮继续
- 原卡片状态实时更新（处理中/已创建/已取消/操作失败）
- 不需要重新发起操作

### 17. 重启后恢复（已完成）
- 脱敏状态文件（公共）+ 私有资源 refs，Hermes 重启后全量恢复
- 文件锁（Windows msvcrt + POSIX fcntl）保障并发安全
- schema_version 字段向后兼容旧格式
- 覆盖看板/详情/催办/批量待办/截止更新 8+ 场景

### 18. 群聊订阅模式（已完成）
- per-group `require_mention` 配置，目标群不需 @ 也能收到消息
- Agent 主动识别群聊目标/承诺/风险/行动项，冒泡"要不要整理成项目？"
- `pilotflow_subscribe_chat` 工具生成配置片段，Agent 引导用户完成

## 评委版创新总结

| 维度 | 一句话 |
| --- | --- |
| 业务创新 | 把群聊里的临时讨论变成可确认、可执行、可追踪的项目启动流程。 |
| 技术创新 | Agent 主驾驶（5 类语义参数必须 LLM 显式传）+ confirm token 幂等 + opaque card ref 安全 + 重启恢复（脱敏 state + 私有 refs + 文件锁）。 |
| 可信控制 | 确认门控 + idempotency key + 高危动作二次确认 + 卡片失败重试 + 全链路脱敏。 |
| 生态融合 | 纯 Hermes 插件（不改源码）+ lark_oapi SDK 直连 + memory/cron/dispatch + 群聊订阅。 |
