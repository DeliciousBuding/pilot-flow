# PilotFlow Demo 截图清单

> 答辩和提交材料需要的关键截图。每个截图对应评分标准的一个维度。

## 必须截图（5 张）

### 1. 产品闭环全景 — 对应评分维度1（完整性与价值）

**画面**：飞书群聊中，PilotFlow 发送的执行计划卡片 + 下方的交付总结消息

**说明**：展示从一句话需求到完整交付的闭环。卡片应显示项目目标、成员、交付物、截止时间。

**获取方式**：
```bash
npm run pilot:run -- --live --confirm "确认执行" --send-plan-card --send-entry-message --pin-entry-message --send-risk-card
```

### 2. 飞书产物矩阵 — 对应评分维度1（流程闭环）

**画面**：飞书文档、多维表格、任务列表三个界面拼图

**说明**：展示 PilotFlow 一次性创建的全套飞书产物。多维表格应显示负责人、截止时间、风险等级等字段。

**获取方式**：运行后分别打开飞书文档、Base、任务截图，拼成一张图。

### 3. Flight Recorder 可视化 — 对应评分维度3（工程规范）

**画面**：Flight Recorder HTML 视图，展示计划、工具调用、产物、时间线

**说明**：展示运行过程的完整追溯能力。应能看到每个步骤的状态、工具调用结果、产物链接。

**获取方式**：
```bash
npm run pilot:recorder -- --input tmp/runs/latest-manual-run.jsonl --output tmp/flight-recorder/latest.html
# 用浏览器打开 HTML 文件截图
```

### 4. 风险裁决卡 — 对应评分维度1（AI 关键作用）

**画面**：飞书群聊中的风险裁决卡片，显示检测到的风险和操作按钮

**说明**：展示 AI 自动识别风险并发起裁决的能力。

**获取方式**：在 live 运行中，风险裁决卡会自动发送到群聊。

### 5. 确认门控交互 — 对应评分维度2（创新性）

**画面**：飞书群聊中的执行计划卡片，带"确认执行"按钮，以及用户确认后的执行结果

**说明**：展示 PilotFlow 的业务级确认机制——先预览再确认，AI 不会未经允许创建产物。

**获取方式**：分两步截图——先截计划卡片，再截确认后的交付总结。

## 截图规范

- 分辨率：至少 1920x1080
- 隐藏敏感信息：API key、token、个人 open_id
- 飞书截图用深色模式或浅色模式均可，但同一组截图保持一致
- 截图命名：`screenshot_01_product_loop.png`、`screenshot_02_artifacts.png` 等
- 存放位置：`materials/screenshots/`（不进 git）

## 截图与评分标准对应

| 截图 | 评分维度 | 评分点 |
| --- | --- | --- |
| 产品闭环全景 | 维度1（50%） | 流程闭环、Demo 稳定 |

| Flight Recorder | 维度3（25%） | 工程规范、可追溯 |
| 风险裁决卡 | 维度1（50%） | AI 关键作用 |
| 确认门控交互 | 维度2（25%） | AI 创新点 |
