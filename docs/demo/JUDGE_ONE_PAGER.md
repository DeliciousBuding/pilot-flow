# PilotFlow — 飞书群里的 AI 项目运行官

> 飞书 AI 校园挑战赛 · 唐丁 · 湖南大学计算机科学与技术 · 2028届

## 一句话定位

**像一个项目经理一样，在飞书群里推动团队从讨论走向交付。**

## 解决什么问题

团队在飞书群里讨论项目，讨论完了信息就散了——目标、负责人、截止时间、风险都没落地。手动创建文档、表格、任务需要 30+ 分钟，容易遗漏。

## PilotFlow 做什么

用户在群里说一句项目需求，PilotFlow 自动完成：

```
群聊意图 → 生成计划 → 人工确认 → 创建文档/表格/任务/风险卡/入口 → 交付总结
```

全程有确认门控，AI 不会未经允许创建产物。每次运行有完整日志可追溯。

## 效率对比

| 步骤 | 手动操作 | PilotFlow |
| --- | --- | --- |
| 整理讨论要点 | 人工翻聊天记录 | 自动提取 |
| 写项目文档 | 10-15 分钟 | 秒级生成 |
| 建多维表格 | 手动建表填字段 | 自动写入 |
| 创建任务 | 逐个创建分配 | 批量创建 |
| 项目入口 | 手动整理链接 | 自动发送并固定 |
| **合计** | **30+ 分钟，多工具切换** | **一句话触发，全程自动** |

## 为什么不是 OpenClaw / 飞书妙记 / 飞书项目

| 维度 | OpenClaw | 飞书妙记/项目 | PilotFlow |
| --- | --- | --- | --- |
| 定位 | 通用 Agent 基础设施 | 会议纪要/项目空间 | 群聊项目运行官 |
| 入口 | 个人助手 | 会议/工作台 | 飞书群聊 |
| 确认机制 | 底层命令审批 | 无 | 项目语义级审批 |
| 运行追溯 | 工程级 trace | 无 | 业务级审计 |

## 技术架构

```
单 Agent + 确认门控 + 工具路由器 + 运行记录
```

- Agent 架构 + 工具注册表，可扩展新工具
- 165 个测试用例全部通过
- 支持接入真实 LLM（OpenAI 兼容接口）
- 飞书原生：IM、卡片、文档、多维表格、任务、入口消息
