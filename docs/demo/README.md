# 演示材料

## 演示脚本

### 场景一：一句话创建项目空间（~30秒）

**演示步骤：**

1. 打开飞书群聊
2. 输入：`@PilotFlow 帮我准备答辩项目空间，成员唐丁，交付物是项目简报和任务清单，5月7日截止`
3. PilotFlow 识别「答辩」模板，自动建议交付物和截止时间
4. PilotFlow 展示执行计划（中文，无技术细节）：
   ```
   📋 执行计划
   - 目标：准备答辩项目空间
   - 成员：唐丁
   - 交付物：项目简报、任务清单
   - 截止时间：2026-05-07
   ```
5. 用户回复「确认」或点击「✅ 确认执行」按钮
6. PilotFlow 自动创建 5 个产物：
   - 📄 飞书文档（格式化 markdown + @提及成员 + 自动开权限）
   - 📊 多维表格（项目状态台账 + 记录 + 自动开权限）
   - 📋 飞书任务（分配负责人 + 设置截止时间）
   - 📌 项目入口卡片（互动卡片 + 可点击链接 + @成员）
   - 📅 日历事件（截止时间提醒）
   - 🔔 截止提醒（Hermes cron 自动调度）
7. PilotFlow 展示结果摘要（display 格式）：
   ```
   ✅ 项目空间已创建: 答辩项目
   📄 文档: https://feishu.cn/docx/xxx
   📊 状态表: https://feishu.cn/base/xxx
   👥 成员: 唐丁
   📋 任务: 项目简报, 任务清单
   ⏰ 截止: 2026-05-07
   📅 日历提醒已创建
   🔔 截止提醒已设置
   💬 已通知群成员
   ```

### 场景二：查询项目状态（带倒计时）

**演示步骤：**

1. 用户输入：`@PilotFlow 项目进展如何？`
2. PilotFlow 发送项目看板卡片：
   ```
   📊 项目看板
   📌 答辩项目 — 成员: 唐丁 | 截止: 2026-05-07 | 🟡 剩余 4 天 | 进行中
   ```
3. 倒计时色标：🟢 正常（>7天）→ 🟡 注意（4-7天）→ 🔴 紧急（≤3天）→ 🔴 已逾期

### 场景三：多轮项目管理

**演示步骤：**

1. 用户输入：`@PilotFlow 把答辩项目的截止时间改成5月10日`
2. PilotFlow 更新内存注册表 + 多维表格记录，发送通知：
   ```
   📝 项目更新: 答辩项目
   截止时间 → 2026-05-10
   🟡 剩余 7 天
   ✅ 状态表已同步
   ```
3. 用户输入：`@PilotFlow 把李四加到答辩项目`
4. PilotFlow 更新成员列表 + 多维表格，@提及新成员

### 场景四：信息不完整

**演示步骤：**

1. 输入：`@PilotFlow 创建一个新项目`
2. PilotFlow 提示信息不完整，询问目标、成员、交付物
3. 补充信息后，PilotFlow 展示计划
4. 确认后执行

### 场景五：模板识别

**演示步骤：**

1. 输入：`@PilotFlow 帮我准备sprint`
2. PilotFlow 自动识别 sprint 模板，建议：
   - 交付物：需求文档、技术方案、测试用例
   - 工期：14 天
3. 用户确认后执行

## 关键演示点

| 演示点 | 体现能力 |
| --- | --- |
| 一句话触发 | 自然语言意图提取 |
| 执行计划展示 | 结构化计划生成 + 模板识别 |
| 确认门控 | 人工确认后才执行 |
| 5个产物自动创建 | 飞书生态深度融合 |
| @提及成员 | 群成员解析 + 文档/消息 mention |
| 自动开权限 | 权限自管理 |
| 任务分配负责人 + 截止时间 | 任务不再是空壳 |
| 倒计时色标 | 项目看板可视化 |
| 多轮更新 + 状态表同步 | 真正的项目管理（非通知） |
| Hermes memory | 创建项目时沉淀项目模式；历史读取建议在下一轮演示 |

## 预期时间

- 场景一：~30 秒
- 场景二+三：~20 秒
- 完整演示：~60 秒
