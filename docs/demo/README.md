## 演示脚本

### 场景一：一句话创建项目空间（~30秒）

**演示步骤：**

1. 打开飞书群聊
2. 输入：`@PilotFlow 帮我准备答辩项目空间，成员示例成员A，交付物是项目简报和任务清单，5月7日截止`
3. Hermes Agent 理解意图，从用户原话提取 title / goal / members / deliverables / deadline 后调用 `pilotflow_generate_plan` 工具，传入结构化字段（不是工具自己关键词解析）
4. PilotFlow 展示执行计划（中文，无技术细节）：
   ```
   📋 执行计划
   - 项目：答辩项目
   - 目标：准备答辩项目空间
   - 成员：示例成员A
   - 交付物：项目简报、任务清单
   - 截止时间：2026-05-07
   ```
5. 用户回复「确认」或点击「✅ 确认执行」按钮
6. PilotFlow 自动创建核心项目产物：
   - 📄 飞书文档（格式化 markdown + @提及成员 + 自动开权限）
   - 📊 多维表格（项目状态台账 + 记录 + 自动开权限）
   - 📋 飞书任务（分配负责人 + 设置截止时间）
   - 📌 项目入口卡片（互动卡片 + 可点击链接 + @成员）
   - 📅 日历事件（截止时间提醒）
   - 🔔 截止提醒（Hermes cron best-effort 调度）
7. PilotFlow 展示结果摘要：
   ```
   ✅ 项目空间已创建: 答辩项目
   📄 文档: <真实链接，私有材料包>
   📊 状态表: <真实链接，私有材料包>
   👥 成员: 示例成员A
   📋 任务: 项目简报, 任务清单
   ⏰ 截止: 2026-05-07
   📅 日历提醒已创建
   🔔 截止提醒已设置（Hermes cron 可用时）
   💬 已通知群成员
   ```

### 场景二：查询项目状态（带倒计时）

**演示步骤：**

1. 用户输入：`@PilotFlow 项目进展如何？`
2. PilotFlow 发送项目看板卡片：
   ```
   📊 项目看板
   📌 答辩项目 — 成员: 示例成员A | 截止: 2026-05-07 | 🟡 剩余 4 天 | 进行中
   ```
3. 倒计时色标：🟢 正常（>7天）→ 🟡 注意（4-7天）→ 🔴 紧急（≤3天）→ 🔴 已逾期

### 场景三：多轮项目管理

**演示步骤：**

1. 用户输入：`@PilotFlow 把答辩项目的截止时间改成5月10日`
2. PilotFlow 更新内存注册表 + 多维表格记录，发送通知：
   ```
   📝 项目更新: 答辩项目
   截止时间 → 2026-05-10
   🟡 剩余 7 天
   ✅ 状态表已同步
   ```
3. 用户输入：`@PilotFlow 把李四加到答辩项目`
4. PilotFlow 更新成员列表 + 多维表格，@提及新成员

### 场景四：信息不完整（Agent 主驾驶证据）

**演示步骤：**

1. 输入：`@PilotFlow 创建一个新项目`
2. Hermes Agent 看到信息不全，**不会**让 PilotFlow 工具瞎猜——直接在群里追问目标 / 成员 / 交付物 / 截止时间
3. 用户补充信息后，Agent 提取结构化字段调用工具，PilotFlow 展示计划
4. 确认后执行

> 这一段是"Agent 是主驾驶 + PilotFlow 是飞书执行层"的核心证据：缺字段时是 Agent 追问，不是工具用关键词兜底拼凑。

## 关键演示点

| 演示点 | 体现能力 |
| --- | --- |
| 一句话触发 | Hermes Agent LLM 从原话提取结构化字段 |
| 执行计划展示 | 结构化计划 + 用户可视化确认 |
| 信息不全时 Agent 追问 | Agent 主驾驶（工具默认拒绝 fallback 关键词解析） |
| 确认门控 | 人工确认后才执行，高危动作（删成员/归档）必须二次确认 |
| 核心产物自动创建 | 飞书生态深度融合 |
| @提及成员 | 群成员解析 + 文档/消息 mention + 提及 markup 注入清理 |
| 自动开权限 | 权限自管理 |
| 任务分配负责人 + 截止时间 | 任务不再是空壳 |
| 倒计时色标 | 项目看板可视化 |
| 多轮更新 + 状态表同步 | 真正的项目管理（非通知） |
| 卡片失败可重试 | 飞书 API 抖动后用户点同一按钮可恢复 |
| Hermes memory | 创建项目时沉淀项目模式（默认不存成员姓名） |
