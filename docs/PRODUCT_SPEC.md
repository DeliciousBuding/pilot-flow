# 产品规格

## 产品定位

PilotFlow 是飞书群聊中的 AI 项目运行官。它将群聊中的项目讨论自动转化为确认过的计划、文档、任务和状态跟踪。

## 用户承诺

| 承诺 | 说明 |
| --- | --- |
| 一句话启动 | @PilotFlow 说一句需求，自动开始项目流程 |
| 真实产物 | 创建飞书文档、多维表格、任务，不是生成文字 |
| 人工确认 | 每次写入前展示计划，确认后才执行 |
| 权限自管理 | 创建的文档和表格自动开放权限，群成员可编辑 |

## 功能分级

### P0 — 核心功能（已完成）

| 功能 | 说明 |
| --- | --- |
| 自然语言意图提取 | 从用户输入提取目标、成员、交付物、截止时间 |
| 结构化计划生成 | 生成可执行的项目计划 |
| 确认门控 | 展示计划卡片，等用户确认后执行 |
| 飞书文档创建 | 格式化 markdown，@提及成员，自动开权限 |
| 多维表格创建 | 自动创建项目状态台账，写入记录，自动开权限 |
| 飞书任务创建 | 创建带描述的任务 |
| 项目入口消息 | 群内项目导航（@成员 + 文档/表格/截止时间链接） |
| @mention | 文档内 mention_user，消息内 `<at>` 标签 |
| 权限自管理 | 链接可查看 + 群成员自动加编辑者 |
| 确认卡片 | 展示项目计划卡片，等用户回复确认 |

### P1 — 增强功能（进行中）

| 功能 | 说明 |
| --- | --- |
| 风险检测 | 自动识别缺失成员、模糊截止时间等风险 |
| 日历集成 | 自动创建截止时间日历事件 |
| 多轮管理 | 改截止时间、查状态、重新分配成员 |

### P2 — 扩展功能（计划中）

| 功能 | 说明 |
| --- | --- |
| 审批流 | 飞书审批 API 集成，项目变更走审批 |
| Hermes memory | 记住用户项目偏好，越用越聪明 |
| 卡片回调 | 按钮点击实时响应 |

## 用户旅程

```
用户在飞书群 @PilotFlow
  ↓
"@PilotFlow 帮我准备答辩项目空间，成员唐丁，交付物是项目简报和任务清单，5月7日截止"
  ↓
PilotFlow 提取意图，发送确认卡片（展示计划）
  ↓
用户回复「确认」或「可以」
  ↓
PilotFlow 创建：
  - 飞书文档（格式化 + @提及 + 权限 + 编辑者）
  - 多维表格（项目状态台账 + 记录 + 权限 + 编辑者）
  - 飞书任务
  - 群入口消息（@成员 + 文档/表格/截止时间链接）
  ↓
PilotFlow 发送交付总结
```

## 技术约束

- 仅支持飞书平台
- 依赖 Hermes Agent 运行时（LLM + 飞书网关）
- 依赖 lark-oapi SDK（Doc/Task/Bitable 操作）
- 消息发送通过 Hermes registry.dispatch
- 需要飞书应用权限（im, docx, bitable, task, drive, calendar）
