# 产品规格

## 产品定位

PilotFlow 是飞书群聊中的 AI 项目运行官。它将群聊中的项目讨论自动转化为确认过的计划、文档、任务和状态跟踪。

## 用户承诺

| 承诺 | 说明 |
| --- | --- |
| 一句话启动 | @PilotFlow 说一句需求，自动开始项目流程 |
| 真实产物 | 创建飞书文档、多维表格、任务，不是生成文字 |
| 人工确认 | 每次写入前展示计划，确认后才执行 |
| 全程可追溯 | 每步操作有记录，出问题可查 |

## 功能分级

### P0 — 核心功能（已完成）

| 功能 | 说明 |
| --- | --- |
| 自然语言意图提取 | 从用户输入提取目标、成员、交付物、截止时间 |
| 结构化计划生成 | 生成可执行的项目计划 |
| 飞书文档创建 | 调用 API 创建项目 Brief 文档 |
| 多维表格记录 | 写入项目状态到 bitable |
| 飞书任务创建 | 创建带负责人的任务 |
| 项目入口消息 | 群内固定项目导航 |
| 确认门控 | 展示计划，等用户确认后执行 |

### P1 — 增强功能（进行中）

| 功能 | 说明 |
| --- | --- |
| 风险检测 | 自动识别缺失成员、模糊截止时间等风险 |
| 风险裁决卡 | 群内风险确认与处理 |
| 运行记录 | 全流程日志与异常追踪 |
| 交互式卡片 | 用飞书卡片替代纯文本，支持按钮操作 |

### P2 — 扩展功能（计划中）

| 功能 | 说明 |
| --- | --- |
| 多项目空间 | 同时管理多个项目 |
| Worker 预览 | 任务执行前预览效果 |
| 自我进化 | 根据历史项目优化建议 |
| 飞书审批集成 | 项目变更走审批流 |

## 用户旅程

```
用户在飞书群 @PilotFlow
  ↓
"@PilotFlow 帮我准备答辩项目空间，成员delicious233，交付物是项目简报和任务清单，5月7日截止"
  ↓
PilotFlow 提取意图，生成计划卡片
  ↓
用户确认执行
  ↓
PilotFlow 创建：文档 + 多维表格记录 + 任务 + 项目入口消息
  ↓
PilotFlow 发送交付总结
```

## 约束

- 仅支持飞书平台
- 依赖 Hermes Agent 运行时
- 依赖 lark-oapi SDK
- 需要飞书应用权限
